## Что умеет приложение

- Загружает изображения из файла и поддерживает вставку из буфера обмена (`Ctrl+V` / `Cmd+V`).
- Позволяет обрезать изображение по пресетам и в свободном режиме.
- Поворачивает изображение на `90°`.
- Скругляет углы по отдельности.
- Удаляет фон локально в браузере без backend.
- Сохраняет отредактированное изображение в исходном формате по умолчанию.
- Конвертирует изображения в отдельном блоке, независимо от редактора.
- Сжимает изображения в отдельном блоке с тремя разными режимами работы.
- Показывает сводку по файлу: формат, размер, размеры изображения и результат после обработки.
- Поддерживает интерфейс на двух языках: `RU / EN`.

## Основные сценарии

### 1. Редактор

Основной сценарий приложения:

- загрузка изображения
- редактирование
- сохранение результата

Поведение экспорта:

- по умолчанию используется исходный формат загруженного файла, если он известен и поддерживается
- если пользователь явно выбирает другой формат, экспорт выполняется в выбранный формат
- качество настраивается только для `JPG` и `WebP`
- при попытке сохранить прозрачное изображение в `JPG` приложение показывает предупреждение и использует выбранную подложку

### 2. Конвертация

Отдельный блок для преобразования форматов, который не зависит от состояния редактора.

Возможности:

- отдельная загрузка файла
- выбор целевого формата
- настройка качества для `JPG` и `WebP`
- предупреждение о прозрачности для `JPG`
- предпросмотр результата
- сводка по файлу до и после конвертации
- скачивание результата и независимый сброс состояния

### 3. Сжатие

Отдельный блок для оптимизации размера файла, который также не зависит от редактора.

Доступны три режима:

- `size-first` — максимальное уменьшение размера с допустимой потерей качества
- `balanced` — компромисс между качеством и размером
- `quality-first` — осторожное уменьшение размера с приоритетом сохранения качества

Секция сжатия:

- анализирует формат, размер, размеры, наличие прозрачности и тип контента
- не конвертирует прозрачные изображения в `JPG`
- при необходимости уменьшает размеры изображения
- пробует несколько кандидатов экспорта
- показывает предупреждения, если выигрыш минимальный или файл уже достаточно оптимизирован

## Экспорт, конвертация и сжатие

### Экспорт из редактора

- основной экспорт больше не работает как агрессивное сжатие
- по умолчанию сохраняет изображение в исходном формате
- не использует фиктивный параметр качества для `PNG`
- не меняет формат без явного выбора пользователя

### Конвертация

- выполняется в отдельном UI-блоке
- не влияет на текущее изображение в редакторе
- поддерживает прозрачность для `PNG` и `WebP`
- для `JPG` требует явную подложку при прозрачности

### Сжатие

- выполняется в отдельном UI-блоке
- использует кандидатный пайплайн
- сравнивает варианты с оригиналом
- отклоняет результат, если он не даёт реального выигрыша
- честно сообщает о минимальном эффекте или уже оптимизированном файле

## Ограничения и особенности

- Приложение работает как `CSR SPA`.
- Вся обработка изображений выполняется в браузере пользователя.
- Действуют ограничения на входные файлы:
  - размер файла до `25 MB`
  - максимальная сторона до `8000 px`
  - ограничение по общему количеству пикселей
- `ICO` экспортируется в фиксированном размере `256x256`.
- При первом запуске удаления фона браузер дополнительно загружает модель и runtime-файлы.
- Для больших изображений при сильном уменьшении используется аккуратное progressive-resize поведение через `Canvas API`.
